Flujos transversales del sistema Recorridos que cruzan múltiples capas
A Autenticación (2-step MFA)
Browser→ Caddy→ POST /api/auth/login
↳ si MFA activo: needsMfa:true (200 sin cookie)
↳ paso 2 con mfaToken TOTP | backupCode→ validateCredentials→ JWT mfaVerified:true→ Cookie httpOnly

TOTP RFC 6238 (30s ±1 ventana) o backup codes bcrypt cost 10 (one-time use: se remueve el hash del array en DB al matchear). Secreto TOTP cifrado AES-256-GCM. Lockout 5 intentos / 15 min.

B Check-in tiempo real (cámara QR)
AttendanceView→ QrScanner (html5-qrcode)
↳ handleScan valida formato (NOM037: ok · rechaza EMP:)→ POST /api/attendance/check-in→ validateQRToken (HMAC 5min)
↳ DB + AuditLog + /emit→ WebSocket→ Dashboard admins

Token NOM037:<hex>:<epoch>:<hmac> con TTL 5 min. Defense-in-depth: cliente valida formato, servidor valida HMAC + anti-replay. Sub-tabs Escanear / Manual en UI del empleado.

C Cumplimiento NOM-037
Geofence + QR HMAC + MFA TOTP→ API→ AttendanceRecord (isLocked) + AuditLog

Registros inmutables tras cierre. Auditoría completa append-only. Identidad verificada por triple factor.
